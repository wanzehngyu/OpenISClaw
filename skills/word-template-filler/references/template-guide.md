# Word 模板制作指南

本文档说明如何制作与 `word-template-filler` 技能配合使用的 Word 模板。

---

## 基础规则

### 占位符命名

占位符必须使用双花括号 `{{ }}`，内容区分大小写：

```
{{TITLE}}           大写英文 → 替换为论文标题
{{ABSTRACT}}        → 摘要正文
{{KEYWORDS}}        → 关键词
{{SECTION:1}}       → 第1个一级章节（自动填充编号+标题）
{{SUBSECTION:2-1}}  → 2.1节内容（不推荐，直接用 CONTENT:N 更稳定）
{{TABLE:1}}         → 第1个表格
{{CONTENT:1}}       → 第1个文本段落块
{{REF}}             → 参考文献列表
```

### 字符格式继承

**重要**：占位符本身的字符格式（字体、字号、粗体、颜色）会被应用到替换后的内容上。

制作模板时：
1. 将占位符文本设置为期望的样式
2. 例如：`{{TITLE}}` 用"标题1"样式 → 填充后标题自动应用"标题1"格式
3. `{{SECTION:1}}` 用"标题2"样式 → 填充后章节标题自动加粗

---

## 段落样式推荐

| 占位符 | 推荐样式 | 字体 | 字号 | 对齐 |
|--------|---------|------|------|------|
| `{{TITLE}}` | 标题1 / Heading 1 | Times New Roman | 16pt | 居中 |
| `{{ABSTRACT}}` | 正文 | Times New Roman | 12pt | 两端对齐 |
| `{{KEYWORDS}}` | 正文 | Times New Roman | 12pt | 左对齐 |
| `{{SECTION:1}}` | 标题2 / Heading 2 | Times New Roman | 14pt | 左对齐 |
| `{{SUBSECTION:3-1}}` | 标题3 | Times New Roman | 12pt | 左对齐 |
| `{{PARAGRAPH}}` | 正文 | Times New Roman | 12pt | 两端对齐 |
| `{{TABLE:1}}` | 表格 | Times New Roman | 10pt | 居中 |
| `{{REF}}` | 正文 | Times New Roman | 12pt | 左对齐 |

---

## 模板结构示例

```
{{TITLE}}          ← 论文标题（建议用"标题1"样式，居中，加粗，16pt）
{{ABSTRACT}}       ← 摘要段落（正文样式，12pt，两端对齐）
{{KEYWORDS}}       ← 关键词（正文样式，12pt）
─────────────────  ← 可加水平线分隔
{{SECTION:1}}      ← 第1章 导论（标题2样式，14pt，加粗）
{{CONTENT:1}}      ← 导论正文段落...

{{SECTION:2}}      ← 第2章 理论基础
{{CONTENT:2}}      ← 理论基础正文...

{{TABLE:1}}        ← 表1（脚本会替换为三线表）
{{CONTENT:3}}      ← 续文段落...

{{SECTION:3}}      ← 第3章 研究假设
{{SUBSECTION:3-1}} ← 3.1 研究假设（标题3样式，12pt）
{{CONTENT:4}}      ← 假设推导正文...

{{REF}}            ← 参考文献（悬挂缩进格式）
```

---

## 表格占位符

`{{TABLE:1}}` 位置的段落会被整个替换为三线表，**不要**在模板中为占位符设置表格格式。直接放置一个含 `{{TABLE:N}}` 文本的段落即可，脚本会自动创建三线表。

三线表格式：
- 顶线：双线（加粗）
- 底线：双线（加粗）
- 栏目线：单线（细）
- 表注：居中，斜体，小五号

---

## 多栏支持

如果需要双栏布局，在 Word 中将模板对应节设为双栏。脚会在填充时保留分栏设置。

---

## 常见问题

**Q: 占位符无法被识别？**  
A: 检查占位符是否在同一个文本框（run）内。如果占位符被分割到多个 run 中，脚本会只处理包含完整占位符的第一个 run。

**Q: 如何处理子章节？**  
A: 使用 `{{SUBSECTION:N-M}}` 占位符，或更推荐的方式：在 `{{SECTION:N}}` 后直接放置 `{{CONTENT:K}}` 段落，脚本会按顺序填充所有内容块。

**Q: 模板中的页眉页脚会保留吗？**  
A: 是的，python-docx 会保留模板中的所有页眉页脚、页码、域等元素。

**Q: 如何添加作者信息？**  
A: 在模板中预留 `{{AUTHOR}}` 占位符，或直接写入模板（不需要替换的部分无需占位符）。

---

## 参考文献格式

`{{REF}}` 会被替换为悬挂缩进格式的参考文献列表，每条文献：
- 左侧缩进 0.74cm
- 首行左侧 -0.74cm（悬挂缩进）
- 字号 12pt
- 行距 1.5 倍

参考文献内容从 Markdown 的 `## 参考文献` 章节中提取。
